# 记忆智能

## 1. 自动分类

### 分类规则

当需要存储一条记忆时，按以下规则判断分类：

| 关键词 / 模式 | 分类 | 目标文件 |
|---------------|------|----------|
| 账号/密码/token/API key/配置/版本号/IP/域名/端口 | 事实 | memories/facts.md |
| 选型/决定/方案/选择了/为什么用/对比/弃用 | 决策 | memories/decisions.md |
| 偏好/喜欢/总是/从不/习惯/风格/约定 | 偏好 | memories/preferences.md |
| 项目/背景/业务/需求/架构/团队 | 上下文 | memories/context.md |
| 性格/沟通/称呼/身份/角色 | 用户画像 | SOUL.md |
| 今天/完成了/发现了/遇到了/临时 | 每日 | journal/YYYY-MM-DD.md |

如果不确定分类，默认写入 journal/，后续整理时再升级到分类文件。

### 条目 ID 格式

```
[类型首字母]-[YYYYMMDD]-[当日序号]
```

| 类型 | 前缀 | 示例 |
|------|------|------|
| Facts | F | F-20260228-001 |
| Decisions | D | D-20260228-001 |
| Preferences | P | P-20260228-001 |
| Context | C | C-20260228-001 |

当日序号：读取目标分类文件，找到今天已有的最大序号 +1。如果今天没有条目，从 001 开始。

### 重要度判定

| 等级 | 判定规则 |
|------|----------|
| **high** | 用户明确说"很重要/一定要记住"；涉及密钥/生产环境/核心架构；影响多个项目 |
| **medium** | 普通技术知识；工具配置；项目信息；一般性偏好 |
| **low** | 临时信息；一次性任务；可能快速过时的内容 |

## 2. FSRS 简化衰减

### 衰减公式

```
score = importance_weight × (0.5 × recency + 0.5 × frequency)
```

**三个因子**：

```
importance_weight:
  high   → 1.0
  medium → 0.6
  low    → 0.3

recency = max(0.1, 1.0 - days_since_last_access / 90)
  → 90天未访问降到 0.1，之后不再下降

frequency = min(1.0, access_count / 5)
  → 被访问 5 次以上 = 满分 1.0
```

### 计算示例

**条目 A**: importance=high, 上次访问 3 天前, 访问了 8 次
```
importance_weight = 1.0
recency = max(0.1, 1.0 - 3/90) = 0.967
frequency = min(1.0, 8/5) = 1.0
score = 1.0 × (0.5 × 0.967 + 0.5 × 1.0) = 0.98 → 热点
```

**条目 B**: importance=low, 上次访问 60 天前, 访问 1 次
```
importance_weight = 0.3
recency = max(0.1, 1.0 - 60/90) = 0.333
frequency = min(1.0, 1/5) = 0.2
score = 0.3 × (0.5 × 0.333 + 0.5 × 0.2) = 0.08 → 归档
```

### 衰减等级

| 分数 | 等级 | 行为 |
|------|------|------|
| > 0.7 | 热点 | 显示在 INDEX.md 热点区 |
| 0.3 - 0.7 | 正常 | 正常保留 |
| 0.1 - 0.3 | 冷却 | 标记为 [冷却]，整理时提醒用户 |
| < 0.1 | 归档 | 提示用户确认后移入 archive/ |

### 何时计算衰减

- **会话结束时**：对今天访问过的条目更新"最近访问"和"访问次数"
- **每月整理时**：对所有条目计算衰减分，全量刷新 INDEX.md
- **用户说"整理记忆"时**：立刻执行全量衰减检查

### 访问计数规则

当 AI 在会话中**实际使用**了某条记忆（引用其内容来回答问题或做决策），该条目：
- 访问次数 +1
- 最近访问日期更新为今天

仅仅读取 INDEX.md 不算访问具体条目。

## 3. 月度压缩

### 触发条件

- 每月 1 日会话开始时，AI 检测到"新月份"
- 或用户主动说："压缩记忆" / "整理上月" / "月度整理"

### 压缩流程（AI 执行步骤）

1. `ls ~/.persistent-memory/journal/` 获取上月所有日志文件
2. 逐一读取上月日志
3. 提取关键信息，按主题合并去重
4. 生成 `archive/YYYY-MM.md`（压缩摘要）
5. 将有价值的信息升级到 memories/ 对应分类（如果还没有）
6. 对所有 memories/ 条目执行衰减检查
7. 归档低衰减条目（用户确认后）移到 archive/
8. 删除上月 journal 文件（询问用户是否保留原文）
9. 全量重建 INDEX.md

### archive/YYYY-MM.md 格式

```markdown
# YYYY年M月 记忆归档

## 本月摘要
（3-5 句话概括本月主要活动）

## 关键事件
- YYYY-MM-DD: 事件描述
- YYYY-MM-DD: 事件描述

## 升级到长期记忆的条目
- F-YYYYMMDD-NNN → memories/facts.md
- D-YYYYMMDD-NNN → memories/decisions.md

## 归档的冷却条目
（从分类文件中移出的低衰减分条目原文）
```

## 4. 冲突检测

写入新记忆前，AI 应检查：

1. **标题匹配**：是否已存在相同或相似主题的条目
2. **标签匹配**：新条目的标签是否与已有条目重叠度 > 50%
3. **处理方式**：
   - 存在相同主题 → **更新**原条目而非新增
   - 新旧矛盾 → 提示用户：「发现与 [条目ID] 冲突，是替换还是保留两者？」
   - 补充信息 → 追加到原条目内容中
