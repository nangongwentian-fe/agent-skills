# 完整工作流

## 会话开始流程

1. **读取 INDEX.md**（必须，~200 tokens，快速了解记忆全貌）
2. **读取 MEMORY.md**（必须，核心长期记忆）
3. **读取 SOUL.md**（如果存在，了解用户画像）
4. **读取今日 journal**（如果存在）
5. **告知用户**：「已加载你的记忆，共 X 条长期记忆，今日已有 Y 条日志」

## 会话中：写入触发

### 显式触发

| 用户说 | 行为 |
|--------|------|
| "记住xxx" / "记录下来" | 判断分类 → 写入对应文件 → 更新 INDEX |
| "更新记忆中的xxx" | 查找原条目 → 覆盖更新 → 更新 INDEX |
| "删除/忘记xxx" | 查找条目 → 删除 → 更新 INDEX |

### 自动触发

检测到以下信息时，**主动提示用户**是否记录：

| 检测到 | 建议分类 | 提示语 |
|--------|---------|--------|
| API key、token | .sensitive/secrets.md | 「检测到敏感信息，是否记录到安全存储？（建议只记录名称，不记录值）」 |
| 架构决策、技术选型 | memories/decisions.md | 「这个技术决策要记录下来吗？」 |
| 用户偏好、习惯 | memories/preferences.md | 「要把这个偏好记住吗？」 |
| 项目背景、业务逻辑 | memories/context.md | 「要记录这个项目背景吗？」 |
| 账号、配置信息 | memories/facts.md | 「要记录这个配置信息吗？」 |
| 完成重要任务 | journal/YYYY-MM-DD.md | 「任务完成，要记录到今日日志吗？」 |

## 会话中：检索触发

| 用户说 | 行为 |
|--------|------|
| "之前那个xxx" / "上次怎么做的" | 三层检索（详见 retrieval.md） |
| "搜索记忆xxx" | grep 全文搜索 |

## 会话结束流程

1. **检查**本次会话是否产生了值得记录的新内容
2. **写入 journal**：如有新内容，追加到 journal/今日.md
3. **更新访问计数**：本次会话中实际使用过的记忆条目，访问次数 +1，最近访问日期更新
4. **更新 INDEX.md**：重写统计、热点列表、最近新增
5. **月初检测**：如果今天是本月第 1 天，提示用户进行月度压缩（详见 intelligence.md）

## 敏感信息处理

### 绝对不记录
- 真实密码明文
- 银行卡号
- API 真实密钥值

### 可以记录
- 账号名 / ID（不含密码）
- 配置项名称（不含敏感值）
- 技术方案描述

### 自动检测

```bash
grep -E "(sk-|ghp_|eyJh|bearer|password|secret)" ~/.persistent-memory/ -r
```

检测到时提示：
```
检测到敏感信息，是否需要记录？（建议只记录账号名，不记录密钥值）
```

敏感信息存储到 `~/.persistent-memory/.sensitive/secrets.md`，该目录已被 .gitignore 排除。

## 整理检查清单

定期（每次心跳或每周）回顾：

1. 最近 7 天日志有哪些新知识？→ 升级到对应分类文件
2. 有没有待办事项完成？→ 更新状态
3. 技术知识有没有过时？→ 更新或归档
4. 账号信息有没有变更？→ 更新 facts.md
5. 有没有衰减分过低的条目？→ 提示用户确认归档（详见 intelligence.md）
